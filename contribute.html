Get involved

A small project where one contribution makes a real difference

NCDMS is built by a small team alongside clinical work, so help in any of the areas below moves the project forward meaningfully — whether that's a single afternoon reviewing a form, or an ongoing role helping shape the dialysis modules.

Ways to help

Four pathways for contributors

01 · Clinical & domain input

Nephrology and dialysis clinical knowledge

The most valuable contributions often come from clinicians and nurses who can review draft forms and workflows against how the unit actually works.

  • Review draft clinical forms for accuracy and completeness
  • Help prioritise which paper/Google Forms workflows to digitise first
  • Advise on dialysis-specific terminology and local practice

02 · OpenMRS & development

Configuration, forms, and custom modules

Experience with OpenMRS 3, React, or general web development is useful for building out forms, dashboards, and the companion modules for dialysis operations.

  • Build or refine O3 form schemas (JSON-based form definitions)
  • Develop patient chart widgets for lab trends and summaries
  • Work on companion modules: queueing, scheduling, bed dashboards

03 · Data, terminology & standards

Concept dictionary and health data standards

Background in health informatics, terminology standards, or OpenConceptLab is helpful for keeping the underlying data model sound and interoperable.

  • Help source and map concepts via CIEL / OpenConceptLab
  • Review LOINC, ICD-10, and FHIR alignment for new forms
  • Contribute to reporting and registry design

04 · Documentation & testing

Writing things down, and trying things out

Documentation and testing are easy to underinvest in on a small project — contributions here have an outsized impact on whether the system is usable and maintainable long-term.

  • Document setup steps, configuration decisions, and known issues
  • Test forms and workflows from a clinic user's perspective
  • Write user guides for clinic staff

Getting started

How to get involved

The project's code and configuration live in a Git repository. If you'd like to contribute, the best first step is to get in touch via the contact page with a short note on what you're interested in — clinical review, development, data standards, or documentation — and we'll point you to the right starting point and current priorities.
