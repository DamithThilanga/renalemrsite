Where things stand

From server setup to a working clinical record

NCDMS is being built in phases, starting with the foundational OpenMRS deployment and core clinical concepts, then layering on nephrology-specific forms, dashboards, and finally the dialysis operational modules.

Progress

Phases of the build

  • Phase 1 Done

    Server & platform setup

    OpenMRS 3 deployed on a university server via Docker Compose, with automated daily database backups, multi-account access for the core team, and resolution of early deployment issues (reverse proxy configuration, container health).

  • Phase 2 In progress

    Concept dictionary & core configuration

    Gap analysis of OpenMRS concepts against nephrology and dialysis data needs, sourcing missing concepts from CIEL via OpenConceptLab, and configuring provider accounts and roles for clinical staff.

  • Phase 3 In progress

    Nephrology clinical forms

    Custom O3 forms for the clinic visit, starting with lab trend capture — serum creatinine with automatic eGFR calculation (CKD-EPI 2021) and CKD stage classification — and a patient chart summary widget showing lab trends over time.

  • Phase 4 Planned

    Remaining clinical forms

    Migrating the existing Google Forms-based data collection (currently around ten forms covering different aspects of the clinic) into structured O3 forms, prioritising the forms used most often in day-to-day clinic visits.

  • Phase 5 Planned

    Dialysis unit modules

    Companion modules for the dialysis side of the unit: session recording, vascular access tracking, a bed/chair allocation dashboard, and a priority queue for patients.

  • Phase 6 Planned

    Scheduling, notifications & reporting

    Automated appointment scheduling with SMS reminders, and clinic-level reporting and registry support for research and Ministry of Health reporting needs.

A note on scope

This roadmap will change

This is a live clinical informatics project shaped by a real unit's needs, so priorities shift as the team learns more about what's practical and what's most useful day to day. The phases above describe a current best guess at sequencing, not a fixed plan. If you're interested in working on something that isn't listed here, get in touch — see the Contribute page.
